DeviceControl

DeviceControl

Controls client devices or reports to CIC on results of executing client device control.

Some user requests may involve controlling of a client device. If a user request is related to controlling of a client device, you will receive a directive message of DeviceControl namespace. Implement your client to control a device based on a received directive message. After executing device control, send the result to CIC using event messages.

DeviceControl provides the following event and directive messages.

Message name Message type Message description
ActionExecuted Event Send this event message to CIC after your client executes device control.
ActionFailed Event Send this event message to CIC if your client cannot or fails to execute device control.
BtConnect Directive Instructs your client to connect a specified Bluetooth device.
BtDisconnect Directive Instructs your client to disconnect a specified Bluetooth device.
BtStartPairing Directive Instructs your client to start a Bluetooth pairing mode.
BtStopPairing Directive Instructs your client to stop a Bluetooth pairing mode.
Decrease Directive Instructs your client to turn down speaker volume or decrease screen brightness by a default unit.
ExpectReportState Directive Instructs your client to report the current state of the appliance to CIC.
Increase Directive Instructs your client to turn up speaker volume or increase screen brightness by a default unit.
LaunchApp Directive Instructs your client to launch a specified app.
OpenScreen Directive Instructs your client to open a setting screen.
ReportState Event The client should use this message to report the current state of the appliance to CIC.
RequestStateSynchronization Event Send this event message to CIC to acquire current state of other client's devices registered on the user's account.
SetValue Directive Instructs your client to set speaker volume or screen brightness to a specified value.
SynchronizeState Directive Instructs your client to update states of other client's devices registered on the user account.
TurnOff Directive Instructs your client to turn off or disable a specified feature or mode.
TurnOn Directive Instructs your client to turn on or enable a specified feature or mode.

Note!

The DeviceControl API is scheduled to be implemented soon. Its specification is described to define an interface and is subject to change.

Send this event message to CIC after your client executes device control.

Payload field

target string Target to control.
  • "airplane": Airplane mode
  • "app": App
  • "bluetooth": Bluetooth
  • "cellular": Mobile communication
  • "channel": TV channel
  • "flashlight": Flashlight
  • "gps": GPS
  • "powersave": Power saving mode
  • "screenbrightness": Screen brightness
  • "soundmode": Sound mode
  • "volume": Speaker volume
  • "wifi": Wireless LAN
 Yes

BtConnect directive

Instructs your client to connect one of the paired Bluetooth devices. If it is paired with more than one device, the client must have defined criteria to decide which device it will connect to. For example, it can attempt to connect in the order of latest to oldest devices it connected to.

Payload field

None

Remarks

  • Only speaker type Bluetooth devices are supported.
  • Your client must send states of Bluetooth devices to CIC frequently, using a Device.DeviceState object.
  • After your client processes this directive message, it must send the result to CIC, using DeviceControl.ActionExecuted or DeviceControl.ActionFailed event message.

Message example

{
    "directive": {
        "header": {
            "namespace": "DeviceControl",
            "name": "BtConnect",
            "messageId": "0f9950d1-c908-4e02-8c38-8e64e840634c",
            "dialogRequestId": "de0a1fd7-2ef1-4040-9469-3a5dd03ef46b"
        },
        "payload": {}
    }
}

ExpectReportState directive

Instructs your client to report the current state of the appliance to CIC. As soon as the client receives this message, he should use the DeviceControl.ReportState event message to report the current state of the appliance. Afterwards, during the designated time on durationInSeconds field, the client should report the status of the appliance according to the designated cycle on intervalInSeconds field.

Payload field

Field name Type Field description Required
durationInSeconds number Reporting period. During the designated period, report the current state of the appliance. Unit is in millisecond. If this field is missing, report only once as the first time. No
intervalInSeconds number Reporting cycle. At the designated cycle, report the current state of the appliance. Unit is in millisecond. This field is valid only when durationInSeconds field is available. If it is missing, then the field is omitted as well. No

Remarks

  • A DeviceControl.ExpectReportState directive message is received when a DeviceControl.RequestStateSynchronization event message is sent to CIC from other client for synchronization.
  • This directive message is sent through downchannel, not as a response to an event message.

Message example

{
  "directive": {
    "header": {
      "namespace": "DeviceControl",
      "name": "ExpectReportState",
      "messageId": "23bdfff7-b655-46d4-8655-8bb473bf2bf5",
      "dialogRequestId": "3c6eef8b-8427-4b46-a367-0a7a46432519"
    },
    "payload": {
      "durationInSeconds": 600,
      "intervalInSeconds": 60
    }
  }
}

LaunchApp directive

Instructs your client to launch a specified app. To specify an app, it returns a custom URL scheme of the app, a relay page URL or the name of the app.

Payload field

Field name Type Field description Required
target string Information of the target app. It can have app information as follows
  • Custom URL scheme: A custom URL scheme of the target app (for example, "naversearchapp://...")
  • Relay page URL: A relay page URL that launches the app if the app is already installed (for example, "http://naverapp.naver.com/...")
  • App name: The name of the app recognized from user speech (for example, "NAVER App")
 Yes

Remarks

  • If your client cannot or fails to launch a specified app, send the result to CIC using an DeviceControl.ActionFailed event message.

Message example

The client should use this message to report the current state of the appliance to CIC.

To deliver the current state of the appliance to the client's device, include Context information(Context) on an event message.

  • Device.DeviceState

None

SynchronizeState directive

Instructs your client to update states of other client devices registered on a user account. Users can use several clients simultaneously with one user account. For instance, a client type may be an app or a speaker such as exclusive speaker for Clova, Wave. An app-type client can control other speaker-type clients. It receives results of executing client control through DeviceControl.SynchronizeState directive messages and update client states accordingly.

The client may receive the directive message on the following situations.

  • When CIC receives an DeviceControl.ActionExecuted or DeviceControl.ActionFailed event message, it sends a DeviceControl.SynchronizeState directive message to all clients registered on the user account to notify the state change of the client device.
  • When CIC receives a DeviceControl.ReportState event message, it sends a DeviceControl.SynchronizeState directive message to all clients registered on the user account to notify the state change of the client device.

Payload field

Field name Type Field description Required
deviceId string The ID of the client device with an updated state. Yes
deviceState Device.DeviceState object An object containing an updated state of the device. Yes

Remarks

DeviceControl.SynchronizeState The directive message is being broadcast to all clients registered on the user account through downchannel. The message does not contain communication ID(dialogRequestId).

Message example

{
  "directive": {
    "header": {
      "namespace": "DeviceControl",
      "name": "SynchronizeState",
      "messageId": "23bdfff7-b655-46d4-8655-8bb473bf2bf5"
    },
    "payload": {
      "deviceId": "{{CLOVA_OAUTH_CLIENT_ID}}",
      "deviceState": {{Device.DeviceState}}
    }
  }
}

See also

  • DeviceControl.ActionExecuted
  • DeviceControl.ActionFailed
  • DeviceControl.ReportState

TurnOff directive

Instructs your client to turn off or disable a specified feature or mode. For example, this directive message can instruct to turn off Bluetooth on a client device.

Payload field

Field name Type Field description Required
target string Target to control.
  • "airplane": Airplane mode
  • "bluetooth": Bluetooth
  • "cellular": Mobile communication
  • "energysave": Energy saving mode
  • "flashlight": Flashlight
  • "gps": GPS
  • "power": Power state
  • "ring": Ring mode
  • "silent": Silent mode
  • "vibrate": Vibration mode
  • "wifi": Wireless LAN
 Yes

Remarks

  • When turning off or disabling a certain control target, you must follow policies of the client device and execute device control appropriately to various situations. For example, to disable a ring mode, you may have to implement to enter a vibration mode or mute mode depending on the client device.
  • Your client must send states of features or modes to CIC frequently, using a Device.DeviceState context object.
  • After your client processes this directive message, it must send the result to CIC, using DeviceControl.ActionExecuted or DeviceControl.ActionFailed event message.

Caution!

A client device must be turned off when its target field is set to "power".

Message example

{
  "directive": {
    "header": {
      "namespace": "DeviceControl",
      "name": "TurnOff",
      "messageId": "23bdfff7-b655-46d4-8655-8bb473bf2bf5",
      "dialogRequestId": "3c6eef8b-8427-4b46-a367-0a7a46432519"
    },
    "payload": {
      "target": "bluetooth"
    }
  }
}
