Glossary

Terms and abbreviations

Note!

This page is updated on an ongoing basis.

Account Linking

Used by an extension to authenticate a user account to provide a 3rd party service. See Linking user account for more details.

CEK

The abbreviation of Clova Extension Kit

CIC

The abbreviation of Clova Interface Connect

CIC API

The format for event messages and directive messages which are categorized into namespaces by the functionalities of Clova. See CIC API for more details.

CIC message

The event messages and directive messages which are used by a client and Clova Interface Connect to send and receive data.

Client credentials

Credential information obtained by registering a client in Clova Developer Console. It is used to obtain a Clova access token. See Creating Clova access token for more details.

Clova

Clova is an AI (artificial intelligence) platform developed and serviced by LINE. Clova recognizes user's speech input or images, analyzes them, and provides information or service as requested by the user. 3rd party developers, by leveraging the Clova technologies, can make a device or home appliance that provides an AI service. They can also offer their content or services to users through Clova.

Clova access token

A means used by Clova to authorize a client when the client tries to send an event message to Clova Interface Cconnect. See Creating Clova access token for more details.

Clova Developer Console

The web tool that provides the following features for a client device that interacts with the Clova platform or for developers who build Clova extension.

  • Registration of a client device and client credentials
  • Registration and deployment of a Clova extension
  • Registration of an interaction model
  • Statistics related to Clova services

Note!

Clova Developer Console is still under development. Contact your counterpart contact personnel to ask for help with Clova Developer Console.

Clova extension

The web application that provides extended capabilities for Clova to enable users to access 3rd party services such as music, shopping, banking or control their home IoT devices. It is commonly called an extension and the Clova platform currently supports and provides the two types of Clova extensions.

  • Custom extension
  • Clova Home extension

Clova Extension Kit (CEK)

The platform that provides the necessary tools and interfaces for developing a Clova extension. It supports communication between Clova and extension.

Clova Home API

The API used to control IoT devices. It uses the Clova Home extension message format. See Clova Home API for more details.

The API provides a format for event message or directive message which are categorized into namespaces by the functionalities of Clova. See CIC API for more details.

Clova Home extension

The extension that provides a service for controlling IoT devices. See Building Clova Home extension for more details.

Clova Home extension message

A message used by a Clova Home extension when communicating information with Clova Extensino Kit to control IoT devices. See Clova Home extension message for more details.

Clova Interface Connection (CIC)

The platform that serves as an interface to Clova. It supports various types of clients which aim to provide an AI assistant service, including PC/mobile app, mobile device, or home appliances. See CIC overview for more details.

Clova Auth API

The API used by a client to obtain a Clova access token. See Clova Auth API for more details.

Content Template

A standardized format for displaying content sent through CIC. See Content Template for more details.

Context information

Context indicates various states information of a client, using context objects. See Context information for more details.

Context Objects

Objects that display context information. See Context information for more details.

Custom extension

An extension that provides extended capabilities. A custom extension lets you provide 3rd party services such as music, shopping or banking. See Building custom extension for more details.

Custom extension message

A message used by Clova Extension Kit and custom extension to send and receive information. See Custom extension message for more details.

Device discovery

A discovery feature that provides a client device with a list of IoT devices that are registered to a user account. See Device discovery for more details.

Dialogue ID

A dialogue ID is a dialogue identifier for distinguishing contexts of a user's spoken request. Users can have a logical dialogue with Clova and a single dialogue can have multiple messages. Generally, each one-time request have its own unique dialogue ID and a dialogue consisting of multi-turn requests have the same dialogue ID. A dialog ID is created when a client sends an event message to Clova Interface Cconnect. It is also included in a directive message which is returned as a response, and the client uses it to determine which event message this directive message is responding to.

Directive message (Directive)

A message sent from Clova Interface Cconnect to control the behavior of a client. A directive message is used to respond to an event message from a client or to send information to a client under certain conditions.

Downchannel

A downchannel is an HTTP/2 stream used when a client receives a directive message from Clova Interface Cconnect. See Connecting with CIC for more details.

Event message (Event)

A message sent from a client to Clova Interface Cconnect. The message is used to send a user request (speech input) or to notify that a client state has changed.

Extension

Another name for Clova extension.

HTTP/2

The second version of HTTP protocol. It is based on SPDY and developed by Internet Engineering Task Force (IETF). It is the enhanced version of HTTP 1.1, which was standardized in RFC 2068 in 1997. It was presented as a Proposed Standard in December 2014 and approved by IESG as Proposed Standard on February 17, 2015. It was published as RFC 7540 in May 2015.

Intent

The specification that defines different types of user requests. A custom extension requires its own interaction model consisting of a set of intents. See Interaction model for more details.

IntentRequest

A request message used to send analysis details of a user request (Intent) to a custom extension. See Handling custom extension request for more details.

Interaction model

A schema that standardizes each type of request that will be sent to a custom extension. See Interaction model for more details.

LaunchRequest

A request message used to notify that a user has requested to start a particular mode or custom extension. See Handling custom extension request for more details.

Message ID

A message ID is an identifier for distinguishing each message in a dialogue. Assume a dialogue of ordering pizzas and asking weather. The dialogues and messages can be identified as follows.

  • Dialogue 1, message 1: Order pizza.
  • Dialogue 1, message 2: How many pizza do you want to order?
  • Dialogue 1, message 3: Two pizzas.
  • Dialogue 1, message 4: Order has been made.
  • Dialogue 2, message 1: How is the weather today?
  • Dialogue 2, message 2: It's sunny today.

OAuth 2.0

A public standard for delegation of access permission. The protocol allows internet users to grant account access permission to other web services or applications. On the Clova platform, it is used for account linking when a client obtains a Clova access token or a user uses a particular extension. See https://tools.ietf.org/html/rfc6749 for more details.

SessionEndedRequest

A request message used to notify that a user has requested to end a particular mode or custom extension. See Handling custom extension request for more details.

Slot

Information necessary for processing a request declared in an intent. It must be defined together when defining an intent. Clova analyzes a user request and extracts information related to slots. See Interaction model for more details.

User speech example

The list of examples showing how user's spoken request can be put in. You can define several example sentences for each intent. Example sentences are denoted with slots. See Context information for more details.

Session ID

A session ID is a session identifier used by an extension to distinguish contexts of a user request. Generally, each one-time request has a unique session ID, and a particular mode (Freetalk, for example) or multi-turn requests have the same session ID. A session ID is created when Clova Extension Kit sends a user request to an extension. A same session ID is maintained when a request such as LaunchRequest is received or when an extension sets the response.shouldEndSession field to false. See Building custom extension for more details.
