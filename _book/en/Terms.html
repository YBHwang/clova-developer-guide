Terms and abbreviations

Terms and abbreviations

Note!

This page is updated on an ongoing basis.

Account Linking

Used when an extension provides a 3rd party service that requires users to authenticate their account. See Linking user account for more details.

CEK

The abbreviation of Clova Extension Kit

CIC

The abbreviation of Clova Interface Connect

CIC API

REST API which CIC provides to clients. Clients use CIC API to exchange information with Clova.

Client credentials

Credentials obtained by registering a client in the Clova Developer Console. They are used to obtain Clova access tokens. See Creating Clova access token for more details.

Clova

Clova is an AI (artificial intelligence) platform developed and serviced by LINE. Clova recognizes user's speech or images, analyzes them, and provides information or services that users have requested. 3rd party developers, by leveraging the Clova technologies, can make a device or home appliance that provides an AI service. They can also offer their content or services to users through Clova.

Clova access token

A means used by Clova to authorize a client when the client tries to send event messages to Clova Interface Connect. See Creating Clova access token for more details.

Clova Developer Console

A web tool that provides the following features for development of a client device or Clova extension that interacts with the Clova platform.

  • Registering a client device and obtains client credentials
  • Clova extension registration (/DevConsole/Guides/CEK/Register_Extension.md) and deployment
  • Registering an interaction model
  • Viewing statistics related to Clova services

Clova extension

A web application that provides extended capabilities to Clova for more diverse user experience, including the capability of controlling IoT appliances at home or 3rd party services such as music, shopping, and banking. It is commonly called an extension and the Clova platform currently supports and provides two types of Clova extensions. To regular users, the extension is refered to as "an extension feature".

  • Custom extension
  • Clova Home extension

Clova Extensions Kit (CEK)

A platform that provides tools and interfaces for development and deployment of a Clova extension. It supports communication between Clova and extensions.

Clova Home extension

An extension that provides a service for controlling IoT appliances. See Building Clova Home extension for more details.

Clova Home extension message

Messages used by Clova Home extensions when they exchange information with Clova Extensions Kit to control IoT appliances. See Clova Home extension message for more details.

Clova Interface Connect (CIC)

A platform that serves as an interface between Clova and a client aiming to provide AI assistant services, such as PC/mobile apps, mobile devices or home appliances. See CIC overview for more details.

Clova Auth API

An API used by clients to obtain Clova access tokens. See Clova Auth API for more details.

Content Template

Standardized formats for displaying content returned from CIC. See Content template for more details.

Context objects

Objects that express current [context information] of a client(#Context). See Context information for more details.

Context information

Context indicates various states of a client. It is expressed with context objects. See Context information for more details.

Custom extension

An extension that provides extended capabilities. A custom extension lets you provide 3rd party services such as music, shopping, or banking. See Building custom extension for more details.

Custom extension message

Messages used by Clova Extensions Kit and custom extensions to exchange information. See Custom extension message for more details.

Device discovery

A feature that provides client devices with a list of IoT appliances registered to a user account. See Providing device discovery for more details.

Dialog ID

A dialog ID is created every time a user initiates new speech input. It is included when clients send Recognize event messages to Clova Interface Connect. Dialog IDs are also included in directive messages when responses are returned from a server side to map those responses to corresponding event messages. Clients check a dialog ID included in a directive message and determine which directive message is responding to which event message. If the dialog ID at the client side is different from that of the directive message, the directive message must be disregarded. See Dialog model for more details.

Directive message

Messages that sent from Clova Interface Connect to control behavior of clients. Directive messages are used when CIC returns responses to event messages from clients, or when CIC sends information to clients under certain conditions.

Downchannel

A downchannel is an HTTP/2 stream through which clients receive directive messages from Clova Interface Connect. See Connecting with CIC for more details.

Extension

Another name for a Clova extension

Event message

Messages that sent from clients to Clova Interface Connect. Event messages send user requests (speech input) or notify that a client state has changed.

HTTP/2

The second version of the HTTP protocol. It is based on SPDY and developed by Internet Engineering Task Force (IETF). It is the enhanced version of HTTP 1.1, which was standardized in RFC 2068 in 1997. It was presented as a Proposed Standard in December 2014 and approved by IESG as Proposed Standard on February 17, 2015. It was published as RFC 7540 in May 2015.

Intent

An intent is a segment that distinguishes the user requests for a Clova extension to handle. The intent is divided into two: a custom intent and a built-in intent. Before implementing a custom extension, an interaction model consisting of a set of intents has to be defined. See An interaction model for more details.

IntentRequest

A type of request message that sends analysis details of a user request (Intent) to a custom extension. See Handling custom extension request for more details.

Interaction model

A model that defines a rule to convert to the standardized format (JSON) for Custom extension to pass user's voice requests to extension. See [An interaction model](/DevConsole/Guides/CEK/Create_Interaction_Model.md for more details.

LaunchRequest

A type of request message that notifies that a user has requested to start a certain mode or custom extension. See Handling custom extension request for more details.

Message ID

A message ID is an identifier for distinguishing individual messages. All event messages and directive messages have their own message ID.

OAuth 2.0

A public standard for delegation of access permission. The protocol allows internet users to grant account access right to other web services or applications. On the Clova platform, it is used for account linking when clients try to obtain Clova access tokens or users try to use a certain extension. Refer to https://tools.ietf.org/html/rfc6749 for more details.

SessionEndedRequest

A type of request message that notifies that a user has requested to end a certain mode or custom extension. See Handling custom extension request for more details.

Session ID

A session ID is a session identifier used by extensions to distinguish context of user requests. Generally, a one-time request has a unique session ID whereas a particular mode or multi-turn request has a same session ID. A session ID is created when Clova Extensions Kit sends a user request to an extension. A same session ID is maintained when requests such as LaunchRequest are made or when an extension sets the response.shouldEndSession field to false. See Building custom extension for more details.

Slot

Information necessary for processing a request declared in an intent. It must be defined in pair with the intent. Clova analyzes a user request and extracts information specific to slots. See An interaction model for more details.

User speech example

A list of examples showing how user's spoken request can be put in. You can define several example sentences for each intent. Example sentences are denoted with slots. See An interaction model for more details.
